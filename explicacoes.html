h1>JavaScript_ECMAScript6

ECMA: especificação da linguagem de programação que é padronizada pelo ECMA262 da ECMA international, servindo como base para futuras implementações de outras linguagens como js, jscript e actionscript

Babel: é um compilador de JS

JS é uma linguagem interpretada: código é executado de cima pra baixo e o resultado desta execução é imediatamente retornado, vc não precisa transformar seu código em algo diferente do executado. Linguagens compiladas precisam transformadas em algo diferente pra que seja executado no computador. Ex.: C e C++

É uma linguagem de tipagem fraca: não há verificação nas operações de JS. É possível utilizar o operador de soma com números ou strings sem que ocorra erro.

É uma linguagem de tipagem dinêmica: não precisamos explicitar o tipo da variáavel no momento da criação

TypeScript: super set da linguagem JS: consegue adicionar tipos em tempo de desenvolvimento e adicona funcionalidades que o JS não tem por padrão. Adiciona interfaces, enum, tipo generic, etc.

https://www.typescriptlang.org/ > playground permite que vc escreva em TS e transforme em JS

Flow:https://flow.org/ > é um outro checador de tipagem mas não é um super set, então não tem enum, interface, executado

Suporte do TS é melhor do que do flow

Funções de primeira classe e ordem maior: Continuar nos 19 min

Currying: técnica de transformar uma função com n parâmetros em apenas uma função que recebe um parâmetro e para cada parâmetro a gente retorna uma nova função

Hoisting:levantar ou suspender algo, é o que acontece no js qdo declaramos variáveis e/ou function; impede que vc use uma var antes de ser definida.

Pode ser dividido em dois tipos: de variáveis e de Funções.
Hoisting de variável só eleva a criação de variável e não sua atribuição.
Hoisting de função a função ser elevada ao topo como um todo

Imutabilidade: em linguagens funcionais os dados que vamos criando nunca mudam; a var nunca vai mudar, se vc precisar mudar ela, vc vai ter que criar uma nova

Ex.: ao invés de mudar um array, iremos criar um novo array, baseado neste que queremos alterar e se formos adicionar um um novo item, pegaremos todo o conteúdo deste array e concatenaremos um novo. se for remover, a gente filtra pra remover

Mesma coisa com objetos: nunca atualizamos propriedades de objetos, a gente copia e altera só o que queremos

Toda vez que passamos um objeto ou array para função, é passado por referÊncia. Se vc alterar eles, vc altera o local que a variável aponta Ele aponta para o mesmo endereço de memória

String literal: '${user.name)} ${user.lastName}'

Tipos e variáveis: existem 3 formas de armazenarmos valores : var, let e const.Let e const introduziram o conceito de escopo de bloco. Quando vc faz um if, for, while, etc, (toda vez que vc usa as chaves) é um escopo de bloco
var nameVar ='Guilherme'; Só aceita escopo global ou de função
let nameLet = "Guilherme"; Aceita escopo de bloco
const nameConst = "Guilherme"; Comportamento paarecido com do let
