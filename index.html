h1>JavaScript_ECMAScript6

ECMA: especificação da linguagem de programação que é padronizada pelo ECMA262 da ECMA international, servindo como base para futuras implementações de outras linguagens como js, jscript e actionscript

Babel: é um compilador de JS

JS é uma linguagem interpretada: código é executado de cima pra baixo e o resultado desta execução é imediatamente retornado, vc não precisa transformar seu código em algo diferente do executado. Linguagens compiladas precisam transformadas em algo diferente pra que seja executado no computador. Ex.: C e C++

É uma linguagem de tipagem fraca: não há verificação nas operações de JS. É possível utilizar o operador de soma com números ou strings sem que ocorra erro.

É uma linguagem de tipagem dinêmica: não precisamos explicitar o tipo da variáavel no momento da criação

TypeScript: super set da linguagem JS: consegue adicionar tipos em tempo de desenvolvimento e adicona funcionalidades que o JS não tem por padrão. Adiciona interfaces, enum, tipo generic, etc.

https://www.typescriptlang.org/ > playground permite que vc escreva em TS e transforme em JS

Flow:https://flow.org/ > é um outro checador de tipagem mas não é um super set, então não tem enum, interface, executado

Suporte do TS é melhor do que do flow
